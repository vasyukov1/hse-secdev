# Check-list Code Review (для Media API)

- [ ] **Объём PR разумный, цель понятна** (реализованы только эндпоинты для Media)
- [ ] **Тесты есть и проходят**: позитивные и негативные кейсы (валидные/невалидные данные, ошибка 404)
- [ ] **Валидация данных реализована**:
  - name: непустая строка, длина 1–255
  - year: >1800 и ≤2050
  - kind: только film / course
  - status: только planned / watching / completed
- [ ] **Ошибки обрабатываются корректно**:
  - клиент не видит stack trace
  - возвращаются правильные HTTP-коды (201, 400, 404, 500)
- [ ] **SQL-запросы защищены** (используются параметризованные запросы / ORM, без конкатенации строк)
- [ ] **Структура и стиль кода понятны** (читаемые имена, логичная разбивка по модулям)
- [ ] **Документация обновлена**:
  - docstrings к методам
  - OpenAPI схема/Swagger отображает новые эндпоинты
- [ ] **CI зелёный** (сборка + тесты проходят)
- [ ] **Добавлена база данных**
